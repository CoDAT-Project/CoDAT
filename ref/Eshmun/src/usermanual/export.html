Exporting and Importing

You can export any model into text representation or an image, the text representation doesn't perserve format / spacing. However, it is useful for use across different versions or exchange over emails.


To export a structure, go to File Menu -> Export -> Image / Text representation (Ctrl + i, Ctrl + t respectively).


To import a structure, go to File Menu -> Import -> Text representation (Ctrl + t), this imports a structure into a new instance of Eshmun, old unsaved content will be lost.


Text Format

Single Kripke Structure:

  • The first line contains the structure name.
  • The second line is "States:", the lines following it contain the states definitions, each on its own line.
  • A State definition is on the form {Name}:{Prop1},{Prop2},...:{startstate}:{retainstate};
  • After the state definitions, there should exist a line with "Transitions:". Following that line the transitions definitions must appear, each on its own line.
  • A Transition definition is on the form {Name_of_start_state}:{Name_of_target_state}:{retaintransition}:{action_name};
  • The specifications are defined after the transition definitions. There should be a line with "Specifications:". The specification formula is written on the next line as text.


Concurrent Kripke Structure:

  • First line is "***MULTI***"
  • After that line, each Kripke Structure is written, in the same format discussed above.
  • The name of each Kripke Structure should have attached to it The pair process inside that structure, comma seperated and between parenthesis.
  • After Each Kripke structure, there should be a line with "***END***".


All formats discussed above are case-sensetive.
